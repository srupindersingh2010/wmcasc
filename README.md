# WMCA scrutiny document library

A GitHub Pages-ready website that downloads WMCA scrutiny meeting papers, minutes, agendas and work programme documents, then publishes a searchable website and a collective work programme spreadsheet.

## Committees covered
The seed list is in `committees.csv` and currently includes the five WMCA committees from the supplied CSV.

## How it works
1. `scripts/scrape_wmca.py` reads `committees.csv`.
2. It visits each WMCA ModGov meeting-list page, follows meeting-document pages, and downloads matching agenda, minutes, report pack and work programme links.
3. It writes:
   - `public/data/documents.json`
   - `public/data/work_programme_items.json`
   - `public/data/work_programme_items.csv`
   - `public/data/work_programme.xlsx`
   - files under `public/documents/`
4. The static site in `public/` displays the data.

## Run locally
```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
pip install -r requirements.txt
python scripts/scrape_wmca.py
python -m http.server 8000 --directory public
```
Open `http://localhost:8000`.

## Host on GitHub Pages
1. Create a new GitHub repository and upload these files.
2. In the repository settings, enable **Pages** and deploy from the `public/` folder, or use a Pages workflow of your choice.
3. Run **Actions → Update WMCA scrutiny documents → Run workflow**.
4. The workflow will refresh the downloaded papers and spreadsheet weekly.

## Notes
- WMCA pages are generated by ModGov and can change markup. The scraper is intentionally pattern-based and stores raw source URLs in the outputs for checking.
- Extraction of work programme item titles is best-effort. PDF and Word files are text-extracted where possible; scanned PDFs may need OCR outside this project.
